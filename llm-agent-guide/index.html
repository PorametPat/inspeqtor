    LLM & Agent Development Guide for Inspeqtor

    Project Overview

    Inspeqtor is a data-efficient framework for characterization and calibration of quantum devices. The project aims to extract maximum information from expensive quantum experiments using advanced statistical and machine learning techniques.

    Key Features

    • Quantum Device Characterization: Tools for analyzing and calibrating quantum hardware
    • Bayesian Optimization of Experiments (BOED): Optimal experimental design for quantum systems
    • Probabilistic Models: Bayesian neural networks for quantum state tomography
    • Physics-Informed Learning: Integration of quantum physics constraints with ML models

    Development Environment Setup

    Prerequisites

    • Python 3.11+
    • uv package manager (recommended) or pip
    • JAX ecosystem familiarity (JAX, Flax, Optax, NumPyro)

    Installation for Development

    # Clone the repository
git clone https://github.com/PorametPat/inspeqtor.git
cd inspeqtor

# Install with uv (recommended)
uv add . --editable --dev
uv sync --all-extras

# Or with pip
pip install -e ".[dev,docs]"

    • Control Sequences: Time-dependent quantum control pulses
    • Expectation Values: Quantum measurement outcomes

    from inspeqtor.experimental.data import QubitInformation, Operator, State

# Quantum operators and states
op_x = Operator.from_label("X")  # Pauli-X

# Neural network models

# Model utilities
expvals = models.observable_to_expvals(observables, unitaries)

    3. Probabilistic Models (probabilistic.py)

    from inspeqtor.experimental import probabilistic as prob
import numpyro.distributions as dist

# Bayesian neural networks
graybox_model = prob.make_flax_probabilistic_graybox_model(
    name="graybox",
    base_model=base_model,
    adapter_fn=models.observable_to_expvals,
    prior=dist.Normal(0, 1)

# SVI training
svi_result = prob.SVIRunResult(params, state, losses, eval_losses)

    4. Physics Simulations (physics.py)

    from inspeqtor.experimental import physics

# Hamiltonian evolution
hamiltonian = lambda t, y: H_static + control_signal(t) * H_drive

unitary = solver(control_params)

# Bayesian optimal experimental design

    key, model, guide, designs, unitaries,

    num_particles=1000

    Common Development Patterns

    1. Working with Control Parameters and Unitaries

    import jax.numpy as jnp

# Control parameters: shape (batch, features)
control_params = jnp.linspace(0, 2*jnp.pi, 100).reshape(-1, 1)

# Unitaries: shape (batch, 2, 2) for qubits
unitaries = jax.vmap(physics_simulator)(control_params)

# Expectation values: shape (batch, n_observables)
expvals = model(control_params, unitaries)

    2. Probabilistic Model Training

    from numpyro.infer import SVI, TraceMeanField_ELBO
import numpyro.optim as optim

# Create model and guide
model = prob.make_probabilistic_model(graybox_model)
guide = prob.auto_diagonal_normal_guide(model, ...)

# SVI setup
svi = SVI(model, guide, optim.Adam(0.01), TraceMeanField_ELBO())
svi_state = svi.init(key, control_params, unitaries, observables)

# Training loop
for i in range(num_steps):
    svi_state, loss = svi.update(svi_state, ...)

    3. Data Processing Pipeline

    from inspeqtor.experimental import data, utils

# Load experimental data
dataset = data.ExperimentalDataset.from_file("data.json")

# Feature engineering
features = utils.polynomial_feature_map(control_params, degree=3)

# Convert measurements to eigenvalues
eigenvals = utils.expectation_value_to_eigenvalue(measurements, shots=1000)

    Testing Guidelines

    Test Structure

    import pytest
import jax.numpy as jnp
import chex
from inspeqtor.experimental import models

def test_model_output_shape():
    """Test that model produces expected output shapes"""
    model = models.linen.WoModel([64], [18])
    params = model.init(key, inputs)
    output = model.apply(params, inputs)

    expected_shape = (batch_size, 18)
    assert output.shape == expected_shape

def test_expectation_values_range():
    """Test that expectation values are in valid range [-1, 1]"""
    expvals = models.observable_to_expvals(observables, unitaries)
    chex.assert_trees_all_close(
        jnp.clip(expvals, -1, 1), expvals, atol=1e-6
    )

    Common Test Patterns

    • Shape testing: Verify tensor shapes throughout pipeline
    • Numerical testing: Use chex.assert_trees_all_close for float comparisons
    • Property testing: Verify physical constraints (e.g., unitary matrices, probability ranges)
    • Integration testing: Test full workflows end-to-end

    Common Issues & Solutions

    1. JAX Compilation Issues

    # Problem: Dynamic shapes causing recompilation
# Solution: Use fixed shapes or static_argnums

@jax.jit
def my_function(x, static_param):
    # Use static_argnums for non-array arguments
    pass

# Better:
my_function = jax.jit(my_function, static_argnums=(1,))

    2. NumPyro Model Debugging

    # Use trace to debug probabilistic models
from inspeqtor.experimental.probabilistic import get_trace

trace = get_trace(model)(control_params, unitaries)
print(trace.keys())  # See all random variables

    3. Memory Management

    # Clear JAX compilation cache if memory issues
jax.clear_caches()

# Use gradient checkpointing for large models
from jax import checkpoint
output = checkpoint(large_function)(inputs)

    Best Practices for LLM/Agent Development

    1. Code Organization

    • Follow existing module structure
    • Use type hints consistently
    • Document function signatures and shapes
    • Keep functions pure when possible (JAX requirement)

    2. Testing Strategy

    • Write tests for new functions immediately
    • Test edge cases and error conditions
    • Use property-based testing for mathematical functions
    • Mock expensive operations (quantum simulations)

    3. Performance Optimization

    • Profile code with JAX's built-in tools
    • Use jax.jit for performance-critical functions
    • Batch operations when possible
    • Consider memory usage in large-scale experiments

    4. Documentation

    • Include docstrings with parameter shapes
    • Provide usage examples in docstrings
    • Update README and docs for new features
    • Document any physics assumptions or conventions

    Debugging Tips

    1. Shape Debugging

    # Add shape assertions throughout code
def my_function(x, y):
    assert x.shape[0] == y.shape[0], f"Batch mismatch: {x.shape} vs {y.shape}"
    # ... rest of function

    2. Numerical Debugging

    # Check for NaNs and infinities
jnp.isnan(tensor).any()  # Check for NaN
jnp.isinf(tensor).any()  # Check for infinity

# Use chex for comprehensive checks
chex.assert_trees_all_finite(output)

    3. Probabilistic Model Debugging

    # Visualize traces
trace = get_trace(model)(...)
for name, site in trace.items():
    print(f"{name}: {site['value'].shape}")
